Write the differences among Local Storage , Session Storage & Cookie?

Local storage: We can store data as key/value pairs in a web browser using the local storage web storage technique on the client's computer.This data persists even if the user closes the browser, shuts down their computer, or restarts their browser.
We can use the setItem() method to set the data in local storage. This method takes two parameters, key and value. With this method, we can store value with a key. localStorage.setItem(key, value); We can retrieve the data stored in it, we can use the getItem() method. This method takes only one parameter, the key whose value we need to access. localStorage.getItem(key); We can remove the data with the help of the removeItem() method, which is stored in memory about the key. localStorage.removeItem(key);
Session Storage: Session storage is similar to local storage, but the data stored in session storage is only available during the current browser session. The data is stored in key-value pairs and can be accessed using JavaScript. Once the browser window is closed, the data is automatically deleted. Session storage can be useful for storing temporary data that is only needed for a single browsing session.
Cookies: Cookies are small text files that are stored on a user's browser by websites. Cookies can store user preferences, login information, and other data that can be used to enhance the user experience. Unlike local storage and session storage, cookies are sent to the server with every HTTP request, which can make them slower to retrieve. Cookies can also be set to expire after a certain amount of time.

What is JS event loop and how it works?

The JavaScript event loop is a fancy way for JavaScript to handle tasks that take a while to complete. When we run JavaScript, it can only do one thing at a time. But sometimes we need to wait for things like data from the server or user input. If we waited for these things before running the next line of code, our app would freeze up. So instead of waiting, JavaScript uses an event-driven system that goes through a queue of tasks. When one task is done, its corresponding function is added to a stack of things to do. If that function generates more tasks, they are added to the queue too. This keeps going forever, so JavaScript can handle long-winded tasks without holding up the whole app.
